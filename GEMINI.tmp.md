# Gemini-Kit v4: Operational Protocol (Continuous Pro Edition)

## 1. ЭФФЕКТИВНОСТЬ КОНТЕКСТА (Precision Tools)
Используйте MCP-инструменты для минимизации шума и максимизации точности.

### А. Исследование и Синхронизация
- **STEP 1**: `/tldr-tree` или `/scout` для маппинга структуры проекта.
- **STEP 2**: `/tldr-map` для анализа сигнатур функций и классов БЕЗ чтения полных файлов.
- **STEP 3**: `/tldr-impact` перед любым изменением существующей функции.
- **STEP 4**: `./scripts/compound-search.sh` для поиска переиспользуемого кода.
- **SYNC**: В начале сессии — `/status` и `kit_get_project_context`.
- **RULE**: Запрещено использовать `read_file` для файлов >100 строк без предварительного использования `/tldr-map`.

### Б. Разработка и Рефакторинг
- **CHECKPOINTS**: `kit_create_checkpoint` обязателен перед любым изменением.
- **CONTEXT**: Используйте `/tldr-context` для быстрого сбора логики по конкретной фиче.
- **PLANNING**: Используйте `/plan-compound` для создания планов на основе существующего опыта проекта.

- **WORKFLOW**: Для атомарных задач используйте `/cook` (Plan -> Code -> Test -> Review).
- **SECURITY**: Любое изменение, затрагивающее внешние интерфейсы или данные, требует передачи задачи `security-auditor` через `kit_handoff_agent`.

## 2. ГЛУБОКАЯ РАБОТА И АГЕНТЫ (Specialized Work)
- **ПРОАКТИВНОЕ ВЫПОЛНЕНИЕ**: Вы ОБЯЗАНЫ проактивно выбирать и использовать наиболее подходящие инструменты (MCP, слэш-команды, скрипты) и специализированных агентов в соответствии с контекстом ЛЮБОЙ задачи (исследование, дизайн, кодинг, безопасность, менеджмент) БЕЗ дополнительного подтверждения со стороны пользователя.
- **КОНТЕКСТНЫЕ ИНСТРУМЕНТЫ**: Автоматически выбирайте инструмент для сбора данных перед началом работы: `/tldr-context` для кода, `/scout` для структуры или `/research` для технологических вопросов.
- **Специалисты**:
    - **Orchestrator**: Координация сложных многошаговых фич.
    - **Security Auditor**: Обязателен для задач с Auth, API и логированием.
    - **Debugger**: Глубокий анализ багов с использованием `kit_auto_rollback`.
    - **Coder**: Основной агент для написания чистого, протестированного кода.
    - **Researcher/Scout**: Для глубокого изучения технологий и кодовой базы.

## 3. НЕПРЕРЫВНОЕ ОБУЧЕНИЕ (Compound Knowledge)
- **LEARNING**: Любая коррекция пользователя или выявленный паттерн ДОЛЖНЫ быть сохранены через `kit_save_learning`.
- **SOLUTIONS**: После успешного решения задачи — `/compound`.
- **SCORING**: После создания решения запустите `./scripts/score-solution.sh` для оценки его качества.
- **ADR**: Архитектурные решения фиксировать через `./scripts/next-adr-id.sh` и `/adr`.

## 4. ГИГИЕНА И ПРОВЕРКА (Finalization)
Перед завершением Directive или перед Git-пушем:
- **HEALTH**: Запустить `./scripts/audit-state-drift.sh` для проверки соответствия контекста коду.
- **HOUSEKEEPING**: Выполнить `/housekeeping` или `./scripts/pre-push-housekeeping.sh`.
- **DOCS**: Проверить актуальность через `./scripts/check-docs-freshness.sh`.

## 5. ПРАВИЛА КОММУНИКАЦИИ И ТОНА
- **Internal Reasoning**: English. **User response: ТОЛЬКО НА РУССКОМ ЯЗЫКЕ**.
- **Neutral Technical Style**: Никаких "идеально", "безупречно" и прочих эпитетов.
- **FORBIDDEN WORDS**: "золотой стандарт", "идеальный", "совершенный", "безупречный", "гарантированный", "настоящий", "профессиональный", "максимальный", "ультимативный", "кристально чистый".
- **No Excuses**: Не извиняться. Констатировать факт и предлагайте исправление.
- **Justification**: Обоснование изменений только техническими метриками (производительность, безопасность, поддержка).
- **No Fillers**: Без вступлений и пустой вежливости.

## 6. БЕЗОПАСНОСТЬ
- **Secrets**: Блокировка вывода секретов (AWS, OpenAI, DB-strings) — абсолютный приоритет.
- **Commands**: Объяснять влияние разрушительных команд перед запуском.
